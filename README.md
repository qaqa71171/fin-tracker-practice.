**Учебная практика УП.02**  
Тема дипломного проекта: Разработка веб-приложения для управления личными финансами «FinTracker»

---

## 📋 Задание 1. Описание тематики дипломного проекта

### Актуальность темы
В условиях экономической нестабильности и растущей популярности осознанного потребления вопрос управления личными финансами становится особенно острым. Значительная часть населения не ведёт регулярный учёт доходов и расходов, что приводит к спонтанным тратам и отсутствию сбережений. Существующие решения либо требуют платной подписки, либо избыточно сложны, либо имеют непрозрачную политику обработки данных. Проект «FinTracker» направлен на создание простого, безопасного и бесплатного инструмента для ведения личного бюджета.

### Целевая аудитория
- **Студенты и молодые специалисты** (18–25 лет) — контроль первых доходов и расходов.
- **Фрилансеры и самозанятые** (25–40 лет) — планирование бюджета при нерегулярном доходе.
- **Семьи** (25–45 лет) — ведение бюджета для достижения финансовых целей.
- **Пользователи, ценящие приватность** — хранение данных локально.

### Предполагаемые функциональные возможности

**Для неавторизованного пользователя:**
- Просмотр целевой страницы
- Регистрация нового аккаунта
- Аутентификация (вход в систему)

**Для авторизованного пользователя:**
- **Дашборд** — финансовая сводка (баланс, доходы, расходы), графики
- **Управление транзакциями** — добавление, редактирование, удаление доходов/расходов
- **Бюджетирование** — установка лимитов с цветовой индикацией
- **Аналитика и отчёты** — отчёты за период
- **Профиль** — настройки пользователя

### Используемые технологии (предварительно)
- **Фронтенд:** HTML5, CSS3, JavaScript (ES6+), Chart.js
- **Подготовка к бэкенду:** имитация API-запросов через localStorage
- **В перспективе:** React, TypeScript, Node.js, Express, PostgreSQL

### Ожидаемый результат
Полностью функциональное веб-приложение «FinTracker» (MVP) с авторизацией, дашбордом и управлением транзакциями. Фронтенд готов к интеграции с бэкендом через REST API.

### Подготовка к интеграции с бэкендом
На данном этапе реализован полноценный фронтенд с хранением данных в localStorage. Все функции работы с данными (авторизация, транзакции, дашборд) вынесены в отдельные функции, которые на следующей практике будут заменены на реальные fetch-запросы к REST API. Структура кода уже готова к интеграции с бэкендом без переписывания логики.
